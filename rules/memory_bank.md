# Memory Bank Rules

## Назначение
Директория `.kilocode/rules/` является **Memory Bank** проекта. Это долгосрочная память для контекста, архитектурных решений и стандартов.

## Правила ведения
1.  **Обновление**: ОБЯЗАТЕЛЬНО обновляйте соответствующие файлы после успешного завершения любой задачи, вносящей изменения в архитектуру, структуру или процессы. Обновляйте только те части, которые были затронуты в ходе выполнения текущей задачи.
2.  **Сохранение контекста**: Запрещено удалять информацию о компонентах или процессах, которые не относятся к текущей задаче, даже если они кажутся незнакомыми. Если контекст задачи ограничен, изменяйте только то, в чем уверены.
3.  **Момент обновления**: Только когда всё проверено, работает и закоммичено.
4.  **Стиль**: Кратко, технично, по делу. "Для себя" (как заметки инженера). Без лишних вежливостей.
5.  **Язык**: Русский.
6.  **Источник истины**: Единственным источником истины является **код**. Если правило противоречит коду, код (обычно) прав, а правило устарело. Но лучше привести код в соответствие с правилами.

## Структура Memory Bank
*   `architecture.md`: Высокоуровневая архитектура, компоненты, потоки данных.
*   `coding_standards.md`: Стиль кода, соглашения, требования к PR.
*   `tech_stack.md`: Используемые технологии, версии, инструменты.
*   `workflow.md`: Процессы разработки, CI/CD, релизный цикл.
*   `documentation.md`: Правила ведения документации (в коде и внешней).
*   `memory_bank.md`: (Этот файл) Правила работы с самой базой знаний.

## Инструментарий
*   Используйте `code-index-mcp` для проверки соответствия правил реальному коду.
*   Используйте `tavily` для поиска лучших практик ведения документации, если возникают сомнения.
