# Стандарты кодирования (VLMHyperBench)

## Общие принципы
1.  **Async-First**: Все сетевые вызовы (API Wrapper, DB, WebSockets) должны быть асинхронными.
2.  **Strict Validation**: Любой структурированный вывод модели и конфигурации должны валидироваться через **Pydantic v2**.
3.  **Type Hinting**: Обязательное использование аннотаций типов для всех аргументов и возвращаемых значений.

## Документирование кода (Docstrings)
*   **Формат**: Google Style.
*   **Язык**: Английский (для обеспечения доступности в Open Source).
*   **Обязательность**: Все публичные классы, методы и функции должны иметь docstrings.
*   **Содержание**: Описание назначения, аргументы, возвращаемые значения, исключения.

## Структура пакетов
*   **Viral Modularity**: Каждый компонент (метрика, датасет) — это независимый Python-пакет.
*   **Локализация**:
    *   `README.md` (английский) — основной.
    *   `README.ru.md` (русский) — опционально/рекомендуется.

## Тестирование
*   **Unit Tests**: Обязательное покрытие критической логики.
*   **Integration Tests**: Проверка взаимодействия компонентов (Inference -> Eval).

## Версионирование
*   **Git Tags**: Используются для фиксации версий пакетов (v0.1.0).
*   **Semantic Versioning**: Major.Minor.Patch.
