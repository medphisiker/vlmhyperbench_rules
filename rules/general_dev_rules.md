# Memory Bank & General Rules

- **Назначение**: Этот файл и директория `.kilocode/rules/` являются твоим Memory Bank. Пиши сюда кратко, технично и "для себя".
- **Обновление**: ОБЯЗАТЕЛЬНО после успешного завершения любой задачи, вносящей изменения в архитектуру, структуру или процессы.
- **Момент обновления**: Только когда всё проверено и работает.
- **Стиль**: Кратко, по делу, без лишних вежливостей.
- **Язык**: Русский.
- **Источник истины**: Единственным источником истины является **код**. Не предполагай, как что-то устроено или работает.
- **Технологии 2026**:
    - **Python 3.13+**, **uv 0.9.26**, **uv.build**.
    - **Pydantic v2** для всех схем.
    - **Dynamic Dependency Composition** вместо Git Submodules для плагинов.
- **Инструментарий**:
    - Всегда используй `code-index-mcp` для поиска нужных участков кода и уточнения деталей реализации перед принятием решений или внесением изменений.
    - При работе с фреймворками и библиотеками уточняй информацию в актуальной документации через `context7`, учитывая версии, используемые в проекте.
    - Перед принятием архитектурных решений или выбором подходов ищи в интернете лучшие практики с помощью `tavily` и обсуди их с пользователем.
- **RUN_MODE**: Всегда учитывай разницу между `dev` и `prod` (использование `.dev.yaml` оверлеев).
